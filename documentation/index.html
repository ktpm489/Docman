    Docman

    Introduction

    Welcome to the Docman Api. Docman is an Api for managing documents. Implemented with an Admin, Editors and users. It provides all that is essential for a management system, users can create documents complete with roles and privileges. Each document defines its access right and the date which it was created.

    Authentication

    All routes apart from /api/v1/users and /api/v1/users/login require a jwt token to be accessed. The token is generated after a successful login or signup. All tokens are valid for 24 hours. Set the header in your API client with the token provided.

    Authorization: token

    Documents

    Create Document

    Creates a new document

    Request:

    {
  "title": "A title",
  "content": "some content goes here"
  "access": "Public"
}

    Response:

    {
    "message": "Document created succesfully",
    "document": {
        "title": "my document",
        "content": "some content goes here",
        "access": "public",
        "createdOn": "2017-08-07",
        "id": 1
    }
}

    Request

    • Endpoint: POST: api/v1/documents

    Notes

    • Document access level is either public, private or role.
    • Only Editors can create role access documents

    Response

    • Status: 201: Created
    • Body (application/json)

    Administrators

    The administrator has access to all documents created and all users information(expept the password) The administrator has the power to assign the `Editor` role to a user as well as create another administrator.

    Change User Role

    Request:

    {
  "role": "3",
}

    Response:

    {
 {
    "message": "Role changed",
    "user": {
        "id": 5,
        "name": "seyi",
        "email": "seyi@seyi.com",
        "roleId": "3",
    }
}
}

    Request

    • Endpoint: PUT: /api/v1/users/:id/role

    Notes

    • Admin users have a role of 1
    • Users have a role of 2
    • Editors have a role of 3

    Response

    • Status: 200: OK
    • Body (application/json)
